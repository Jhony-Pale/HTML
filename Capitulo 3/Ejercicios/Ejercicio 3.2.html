3.2: Indique si cada uno de los siguientes enunciados es verdadero o falso. Si es falso, explique el por qué.

  1. Cualquier tipo input particular de formulario de HTML5 debe desplegarse de manera idéntica en todos los navegadores compatibles con HTML5.
    R: Falso, pueden variar entre navegadores.

  2. Cuando se coloca el enfoque en el campo de texto (es decir, si el cursor está en el campo de texto), se envía el texto del atributo placeholder al servidor.
    R: Falso, el texto placeholder desaparece y no se envía aunque el usuario haya dado clic en el bot∮n enviar.

  3. No es necesario incluir el atributo autofocus en sus formularios.
    R: Verdadero, no hay problema si se agrega o no.

  4. Los nuevos tipos input de HTML5 tienen validación automática del lado del cliente, con lo cual se elimina la necesidad de agregar secuencias de comandos (script) complicadas a sus formularios para validar la entrada del usuario y se reduce la cantidad de datos inválidos que se envían.
    R: Verdadero, de esta manera se ahorran unas líneas de código.

  5. El tipo input range se valida automáticamente de manera intríseca cuando el navegador lo despliega como un control deslizante, ya que el usuario no puede mover el control fuera de los límites del valor mínimo o máximo..
    R: Verdadero.

  6. HTML5 cuenta con una validación automática para el tipo input tel.
    R: Falso, HTML5 no cuenta con esta validación debido a que el formato de los números telefónicos varía en muchas zonas.

  7. Si el usuario introduce un URL con un formato inapropiado en un tipo input url, no se validará. HTML5 no valida si el URL introducido existe en realidad.
    R: Verdadero.

  8. El elemento nav muestra un menú desplegable de hipervínculos.
    R: Falso, este elemento agrupa vínculos de navegación.

  9. El elemento header puede usarse sólo una vez en la página.
    R: Falso, se puede usar varias veces en una página.

  10. Los elementos nav pueden anidarse en un elemento aside.
    R: Verdadero, si se pueden agrupar.

  11. Podemos usar el elemento brk para evitar dividir palabras en lugares incorrectos.
